动态标绘专题

简介

SuperMap iClient3D for WebGL提供了三维动态标绘模块，支持用户在服务器端发布动态标绘服务，在客户端通过访问服务端发布的标绘服务来实现动态标绘与态势推演功能。用户可以在三维场景中方便灵活的标绘三维点标号、线面标号，并进行态势推演展示。

    • 技术特点：动态标绘技术可以动态的标绘点标号和线面标号，能够独立修改每个标号的风格、位置、形状，并且提供7种动画实现三维态势推演功能。
    • 应用场景：主要用于在三维场景中实现军事作战方案、应急处置方案、公安围捕方案等指挥调度方案的三维动态标绘。通过提供的各类具有特殊意义的图形、模型标号，可以在三维场景上非常直观、准确地展现事件的处置方案，提高事件处置过程中会商和沟通的效率。

使用说明

首先我们得做如下的准备工作：

    • 保证服务器存在可用的标绘服务，或者我们可以自行去服务器服务管理页面，通过关联一个可用的标号库快速发布一个标绘服务。
    • 在cesiumContainer上添加一个 viewer，并在此基础上通过访问由SuperMap iServer发布的三维服务（如地形数据、倾斜摄影数据、精细建筑模型等），在相应的三维场景中进行动态标绘。

一、动态标绘专题-鼠标标绘

1、创建动态标绘总控对象

首先，需要创建一个动态标绘总控对象，并为其指定标绘服务serverUrl与场景scene。 

                    var plotting = cesium.Plotting.getInstance(serverUrl, scene);

2、创建标绘图层

创建一个标绘图层对象,并将标绘图层添加到scene中。

                   var plottingLayer = new cesium.PlottingLayer(scene, "plottingLayer");
                   scene.layers.add(plottingLayer);

3、创建标号库管理器对象

其次，我们需要从动态标绘总控对象中获取标号库管理器对象。

                    var symbolLibManager = plotting.getSymbolLibManager();

4、创建编辑控件和绘制控件

最后，需要创建编辑控件和绘制控件，开始鼠标标绘和编辑。

创建编辑控件

                    var plotEditControl = new cesium.PlotEditControl(scene, plottingLayer);

创建绘制控件

                    var plotDrawControl = new cesium.PlotDrawControl(scene, plottingLayer);

标绘结束，激活编辑控件

                    plotDrawControl.drawFinishEvent.addEventListener(function () {
                        plotEditControl.activate();
                        });

5、初始化标绘面板和属性面板

为了使得用户更加方便的进行标绘和编辑标号，SuperMap iClient3D for WebGL动态标绘提供了标绘和属性两个面板，初始化面板的代码如下：

初始化标绘面板

                    initPlotPanel("plotPanel", serverUrl, plotDrawControl, plotEditControl,plotting);

初始化属性面板

                    stylePanel = new StylePanel('stylePanel', plotEditControl, plotting);

到这里为止，动态标绘初始化工作已经完成，在客户端可以进行鼠标标绘了。

二、动态标绘专题-编程标绘

SuperMap iClient3D for WebGL动态标绘技术中，支持编程实现动态标绘。编程标绘也需要首先创建动态标绘总控、标绘图层、标号库管理器对象、编辑控件与绘制控件，实现方法同“动态标绘专题-鼠标标绘”的第1、2、3、4部分内容，这里不再重复。

1、创建标号

根据标号库编码、标号编码、位置点数组可以创建标号，语句如下：

                   plottingLayer.createSymbol(libID, code, points, function (even) {
                                var geo = even.feature;
                        });
                   // 其中，libID为标号库编码，code为标号编码，points为位置点数组，function (even){}为回调函数。

2、修改标号属性

标号可以修改的属性分两部分存储，点标号独有的属性从geo.symbol里获取,而点标号和线面标号共有的属性从geo.symbolStyle里获取。下面的代码以少部分属性为例，3、标号属性列表具体列出标号的主要属性。

                    geo.scale = new cesium.Cartesian3(1, 1.0, 1.0);  //缩放比例
                    geo. gridLineWidth = 6;  //公告板模式线宽
                    geo.gridSurroundLineWidth = 4;  //公告板模式衬线宽
                    geo.symbolStyle.surroundLineType = 1;  //衬线类型
                    geo.symbolRank = 1;  //标号级别
                    geo.lineSymbolID = 1;  //点标号边线类型

三、态势推演

态势推演的根本目的是迅速、准确地为决策者定下行动决心提供支持。三维由于其形象直观的特点，更是进行态势推演的最佳技术。

1、态势推演动画

SuperMap iClient3D for WebGL 动态标绘技术提供了态势推演过程常用的7种动画：属性、闪烁、路径、旋转、比例、显隐和生长动画。

2、动画管理类

SuperMap iClient3D for WebGL动态标绘技术中态势推演主要是利用GOAnimationManager类（动画管理类）完成态势推演功能。为方便用户顺利实现态势推演功能，下面列出GOAnimationManager类使用方法。

(1)获取动画管理对象

                   animationManager = plotting.getGOAnimationManager();

(2)创建动画

                   animationManager. createAnimation(animationType, name, goFeature);
                   // animationType表示动画类型，name为动画名称，goFeature为动画对象
                   // 注意：在创建动画时，必须选中想要创建动画的对象

(3)执行动画

                   animationManager.execute();

(4)播放动画

                   animationManager.play();
                   // 注意：动画播放时可以单个播放，也可以多个动画一起播放。

(5)暂停动画

                   animationManager. pause ();

(6)停止播放动画

                   animationManager. stop ();

(7)重置动画

                    animationManager. reset();

四、态势图管理

SuperMap iClient3D for WebGL动态标绘技术支持态势图管理，包括保存态势图到服务器、打开态势图文件、删除态势图文件、上传态势图文件等功能。态势图管理的各项功能是通过SitDataManager类来完成的。

首先，我们需要通过下面的语句获取态势图管理对象：

                   sitDataManager = plotting.getSitDataManager();

态势图管理的功能将在下表中列出。

序号

功能描述

实现语句

1

保存当前态势图数据到服务器

sitDataManager . saveSmlFile();

2

打开指定的已发布态势图文件

sitDataManager .openSmlFileOnServer(smlFileName);

3

删除指定的已发布态势图文件

sitDataManager .deleteSmlFileOnServer(smlFileName);

4

上传态势图文件

sitDataManager .uploadSmlFile(fileId, success, fail);
